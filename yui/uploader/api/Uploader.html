Class Uploader - extends Widget

Provides a file uploader widget allows a user to select multiple files or drag and drop files from the desktop into the widget. Image files are optionally resized. All files maybe are optionally archived (zip or tar) and compressed (bzip2, gzip, zip) to save bandwidth and speed uploads. Cross domain uploads are allowed if the destination host has a master policy file which allows uploads from the source domain (crossdomain.xml). Real upload progress is shown to the user.

Uploading to HTTPS is not supported at this time (BrowserPlus Uploader service currently lacks support for it).

Configuration Attributes

archiveFormat - String

Set the optional archive + compression format. When set, all files will be archived and compressed (depending on the format) into a single file before upload. Used in conjunction with image resizing, upload speed should improve with less bits sent over the wire. Valid values: ('tar', 'tar-bzip2', 'tar-gzip', 'zip', 'zip-uncompressed').
Default Value: null

cookie - String

The optional cookie string POSTed to server during upload.
Default Value: null (cookie not sent)

fieldName - String

The name of the variable in the POST request containing the file data. "file" by default.
Default Value: file

files - Array

Array of files that are going to be uploaded. Each file is a object that contains {name, size, TBD}.
Default Value: []

maxFileSize - Number

Maximum size in bytes of a single file. Default is 2MB. Set size to 0 or less for no limit.
Default Value: 2097152 (2MB)

mimeTypes - Array

An array of mimeTypes for which the selected files will be filtered
Default Value: []

quality - Number

The quality of the scaled image (0-100). Lower qualities result in faster operations and smaller file sizes, at the cost of image quality.
Default Value: 80

resizeHeight - Number

Maximum height of an uploaded image. By default, uploaded images are not resized. Set this value to non-zero to restrict the size of an image.
Default Value: 0 (don't scale image)

resizeWidth - Number

Maximum width of an uploaded image. By default, uploaded images are not resized. Set this value to non-zero to restrict the size of an image.
Default Value: 0 (don't scale image)

timeout - Number

An optional integer value between 3 and 240s that places an upper bound in seconds on how long to wait without being able to send or receive data from the server (i.e. while server is processing uploaded content). Default is 240s.
Default Value: 240

uploadUrl - String

The URL of the upload script on the server. If this value is not set, files will not be uploaded.
Default Value: null

vars - Object

The object containing variables to be sent in the same request as the file upload.
Default Value: null
